# Group

## Introduction

Group actions allow users with sufficient permissions to manage groups and invite members. To know
more about the groups visit [this page](../02-guides/groups.md). Group actions are client side actions,
i.e. these actions are not implemented for the server side. Supported group actions are as follow.

- [`group.invite`](#groupinvite)
- [`group.join`](#groupjoin)
- [`group.list`](#grouplist)
- [`group.members`](#groupmembers)

### group.invite<Side name="group.invite" />

This action allows a user with sufficient permissions to invite users to a group. Supported fields
for this action are `type`, `id`, `email`, and `role`. Id is the id of the group to which the user
will be invited, email is the email of the user to be invited to join the group and role is the role
for which the user is being invited. Default value for role is member. Type is the only required
value for using this action. Here is an example of how you can use this action.

```yaml validate blocks-snippet
blocks:
  - type: form
    version: 0.29.11
    parameters:
      fields:
        - name: email
          label: { translate: email }
          type: string
          format: email
          requirements: [required: true]
        - name: id
          type: hidden
    actions:
      onSubmit:
        type: group.invite
        onSuccess:
          type: log
```

### group.join<Side name="group.join" />

This action allows a user to join a group. Supported field for this action is `type`, which is
`group.join` in this case. It returns an error if the user tries to join a group that they are already
a member of, or if the group security is defined as `invite`. Here is an example of how you can use
this action to allow users to join a group selected by them.

```yaml validate blocks-snippet
blocks:
  - type: form
    version: 0.29.11
    parameters:
      fields:
        - name: group
          label: { translate: group }
          requirements: [required: true]
          type: enum
          enum:
            - value: group1
            - value: group2
    actions:
      onSubmit:
        type: group.join
        remapBefore: { prop: group }
        onSuccess:
          type: log
```

### group.list<Side name="group.list" />

This action allows to get a list of all of the groups in an app. Supported field for this action is
`type`, which is `group.list` in this case. An example use case of this action is providing users of
your app the option to choose the group they want to join. Here is an example of previously mentioned
use case of the action.

```yaml validate blocks-snippet
blocks:
  - type: form
    version: 0.29.11
    parameters:
      fields:
        - name: group
          label: { translate: selectGroup }
          type: enum
          action: getGroups
    actions:
      getGroups:
        type: group.list
        remapAfter:
          - array.map:
              object.from:
                label:
                  prop: name
                value:
                  prop: id
```

### group.members<Side name="group.members" />

This action gets a list of a group’s members. Supported fields for this action are `type` and `id`.
Id is the ID of a specific group to get the members from and the type is `group.members`. This can be
particularly helpful in keeping a record of roles assigned to various users. Both the fields are
required. Here is an example of this action being used to fetch all the employees of a particular
department in a company.

```yaml validate blocks-snippet
blocks:
  - type: data-loader
    version: 0.29.11
    actions:
      onLoad:
        type: group.members
        id: { prop: department }
        onSuccess:
          type: log
```
